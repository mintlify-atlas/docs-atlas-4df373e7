Broccoli is a fast, reliable asset pipeline for applications that run in the browser. Unlike traditional build tools that recompile everything on every change, Broccoli models your build as a directed graph of nodes — each node reads from input directories and writes to an output directory — enabling constant-time incremental rebuilds regardless of project size.

## Why Broccoli?

Broccoli's architecture keeps rebuilds fast no matter how your project grows. Each plugin in the graph only re-runs when its specific inputs change, so adding more plugins doesn't slow down your feedback loop.

## Get started

<Steps>
  <Step title="Install Broccoli">
    Install the Broccoli library and CLI tool into your project.

    ```bash
    npm install --save-dev broccoli
    npm install --global broccoli-cli
    ```
  </Step>
  <Step title="Create a Brocfile.js">
    Add a `Brocfile.js` at your project root that exports a function returning a build tree.

    ```js Brocfile.js
    export default () => 'app';
    ```
  </Step>
  <Step title="Build or serve">
    Run a one-time build into a `dist/` directory, or start the development server with live rebuilds.

    ```bash
    broccoli build dist   # one-time build
    broccoli serve        # development server on :4200
    ```
  </Step>
</Steps>

<Tip>
  For TypeScript-based build files, name your file `Brocfile.ts` and install `ts-node`. Broccoli will automatically invoke `ts-node` to compile it.
</Tip>
